Manual de INVERDAN

Guía completa para configurar, entrenar y operar el sistema de trading automático

Índice
1. Credenciales API 2. Instalación 3. Entrenar modelos ML 4. Arrancar el bot 5. Monitorizar señales 6. Activar Auto-Trade 7. Gestión de riesgo 8. Trading real 9. Flujo con GitHub
1
Credenciales API de Alpaca
Conecta el bot con tu cuenta Paper Trading
✓ Completado

Las credenciales se obtienen en Alpaca → Profile Settings → Manage Accounts → Paper Accounts. Se guardan en el fichero .env en la raíz del proyecto.

# /Applications/MAMP/htdocs/inverdan/.env
ALPACA_API_KEY=PKxxxxxxxxxxxxxxxxxxxxxxxxx
ALPACA_API_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
ℹ El sistema usa Paper Trading por defecto — dinero ficticio, sin riesgo real. La cuenta empieza con $100,000 simulados.
⚠ El Secret solo es visible una vez en el dashboard de Alpaca. Si lo pierdes, genera uno nuevo en la misma sección.

3
Entrenar los modelos ML
RandomForest por símbolo · ~5-10 minutos
⏳ Pendiente

El script train.py descarga 1 año de datos históricos por símbolo, calcula todos los indicadores técnicos y entrena un modelo RandomForest para cada uno.

# Entrenar todos los símbolos del config.yaml (recomendado)
python3 train.py

# Entrenar solo algunos símbolos
python3 train.py --symbols AAPL TSLA NVDA

# Reentrenar forzando (aunque ya existan modelos)
python3 train.py --force

# Ver progreso y métricas
python3 train.py --symbols AAPL
ℹ Los modelos se guardan en models/. Repite el entrenamiento cada 2-4 semanas para mantenerlos actualizados con datos recientes.

Arrancar el bot
Dos terminales: bot + dashboard web

El sistema tiene dos componentes que corren por separado:

# Puerto alternativo
./start_dashboard.sh 8080

# Acceder en el navegador:
# http://localhost:5050
ℹ El bot precarga los últimos 5 días de datos históricos al arrancar para que los indicadores tengan contexto suficiente antes de analizar el mercado en tiempo real.
5
Monitorizar señales
Mercado abierto: lunes–viernes 15:30–22:00 hora española
⏳ Pendiente

Con el bot corriendo y el mercado abierto, verás señales en tiempo real en este dashboard. El sistema genera tres tipos:

BUY
Señal de compra. Reglas técnicas y ML detectan condiciones alcistas con confianza ≥65%.
SELL
Señal de venta/bajista. Sobrecompra detectada o tendencia de caída confirmada por ML.
HOLD
Sin señal clara. Reglas y ML no están de acuerdo, o la confianza es insuficiente.
⚠ Observa las señales durante 1-2 semanas sin Auto-Trade activo. Comprueba que tienen sentido antes de dejar que el bot opere por su cuenta.

Qué validar antes de activar Auto-Trade:

  • Las señales BUY y SELL coinciden con movimientos reales del precio
  • El win rate estimado supera el 55%
  • Los indicadores RSI, MACD y BB tienen valores coherentes
  • La confianza media de las señales está por encima del 65%
6
Activar Auto-Trade
El bot ejecuta órdenes automáticamente
⏳ Pendiente

Hay dos formas de activar el trading automático:

Opción A — Desde el dashboard

Ve a Servidor → Control del Bot → toggle Auto-Trade o usa el botón "Cambiar" en la tarjeta del Resumen.

ℹ El bot detecta el cambio en state.json sin necesidad de reiniciarse.
Opción B — Al arrancar
python3 main.py \
  --no-dashboard \
  --auto-trade

Cada operación utiliza órdenes bracket: una orden principal de entrada más stop-loss y take-profit automáticos que viven en los servidores de Alpaca, independientemente de si el bot está corriendo.

8
Pasar a trading real
Solo cuando las métricas paper sean positivas
⚠ Avanzado
⛔ Opera con dinero ficticio al menos 2-4 semanas antes de considerar el trading real. Las pérdidas en live son reales e irrecuperables.

Métricas mínimas recomendadas en Paper Trading antes de pasar a Live:

  • Win rate > 55% durante al menos 50 operaciones
  • Sharpe Ratio > 1.0
  • Max Drawdown < 10%
  • PnL total positivo durante 4+ semanas consecutivas

Cuando estés listo, el proceso es:

# 1. Abre una cuenta Live en Alpaca (requiere verificación KYC)
#    → app.alpaca.markets → "Open Live Account"

# 2. Genera nuevas API Keys Live en Alpaca
#    → Profile Settings → Manage Accounts → Live Accounts

# 3. Añade las keys al .env
ALPACA_API_KEY=TU_LIVE_API_KEY
ALPACA_API_SECRET=TU_LIVE_API_SECRET

# 4. Cambia paper_trading en config.yaml
alpaca:
  paper_trading: false   # ← CAMBIO CRÍTICO

# 5. Reinicia el bot
python3 main.py --no-dashboard --auto-trade
⚠ Empieza con capital mínimo ($500–1.000) y ve escalando gradualmente. Nunca inviertas más de lo que puedas permitirte perder.
9
Flujo de trabajo con GitHub
Guarda y sincroniza los cambios del proyecto
Recomendado

El repositorio está en github.com/alfons-/inverdan. Usa este flujo cada vez que modifiques algo — configuración, código, modelos, etc.

Ciclo diario habitual
# 1. Ver qué ha cambiado
git status
git diff

# 2. Añadir los cambios (revisa uno a uno con -p)
git add -p                     # interactivo, recomendado
git add config.yaml            # o fichero concreto

# 3. Guardar el commit con descripción clara
git commit -m "feat: añadir símbolo META al config"

# 4. Subir a GitHub
git push
Historial y estado
# Ver commits recientes
git log --oneline

# Ver cambios sin confirmar
git diff

# Ver qué está staged
git diff --staged
Desde otro ordenador
# Clonar el proyecto
git clone https://github.com/\
  alfons-/inverdan.git
cd inverdan
cp .env.example .env
# → rellena tus API keys en .env
pip3 install -r requirements.txt
pip3 install alpaca-py

ℹ Los modelos entrenados (models/*.joblib) no se sincronizan — cada instalación nueva debe ejecutar python3 train.py para regenerarlos.
⚠ Nunca hagas git add .env. El fichero .env contiene tus API keys reales y está excluido por el .gitignore. Si lo subes accidentalmente, regenera las keys en Alpaca inmediatamente.
